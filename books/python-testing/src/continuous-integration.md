# Continuous Integration (CI)

There are various service (e.g. GitHub Actions, GitLab Piplines) that allow you (the administrator of a project) to run arbitrary code every time someone pushes out changes to GitHub or GitLab (Etc.)
You can configure various checks there, for example you can run all the tests of the project or you can run linters on the project.


* Reduce feedback cycle.
* Avoid regression.
* On every push.
* It can run every few hours full test coverage.
